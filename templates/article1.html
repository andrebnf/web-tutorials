Intro

HTML is one the languages we use to develop websites. The acronym HTML stands for Hypertext Markup Language.

HTML is the base language for the web. It was created in order to be easily comprehended by human beings and also for machines, as for example Google or other systems that search the web capturing information.

Source: Tableless

The page you're looking at right now is HTML (and its style, which we'll come along soon) rendered by your browser.

The HTML syntax is easy to understand. To start with, everything is a tag. In the example below, we can see a simple Hello World HTML code, which implements the essentials for a HTML code to run.

<!-- This is a comment -->
<!doctype html> <!-- This line tells the browser we are using HTML5 validation -->
<html>
<head> 
  <title>Page Title</title>
</head>
<body>
  <h1>Hello Web</h1>
</body> <!-- A good HTML ninja always close his tags! -->
</html>

Obs: It's essential for you to understand that the browser will run and display a HTML file that contains one or more errors. Therefore, the programmer has the responsability to write a clean and flawless code, following the HTML specification from W3C.

HTML Semantics

The HTML language contains semantic and non-semantic tags. Semantic means "meaning". In HTML, meaning is important both for the developer and the browser.

At earlier versions of HTML, you would define a header by writing:

<div class="header">
  <h1>Some Title</h1>
</div>

For the browser, this means that there is a header text inside a div, but it can't exactly define wheter this header text actually is on the page.

A more meaningful way to do it with HTML 5 is:

<header>
  <h1>Some Title</h1>
</header>

Wow! It's quite a change! So now, the browser knows you have a title inside a header and it even knows you have a header.

Below, you can check some example of semantics in HTML:

<!-- Non-semantic tags (tells nothing about its content): -->
<div> and <span>
<!-- Semantic tags (clearly defines its content): -->
<form>, <table>, and <img>
<!-- Some of the new semantic tags in HTML5: -->
<article>, <aside>, <header>, <nav>, and <section>

To check all the new semantic tags added on HTML5, check this W3C article.

The Document Object Model (DOM)

This is the HTML structure itself. All the HTML can be represented in a tree, and that's what the Object model is. That means that one element can have children, grandchildren and so on.

When your browser interprets your HTML code, it builds up a tree, having the element and it's children. Once we have this tree, it's easy to refer a certain object in the document. Say we have a <header>; this element is on the DOM tree and the browser knows exactly where it is (and obviously its children too).

You'll realize how important this concept is when we get to stylesheets. For now, let's focus on HTML.

The Importance of Semantic HTML in SEO (Search Engine Optimization)

If you're programming for the web, you will want Google or any other Search Engine to find your page. It's easier to find things which actually have a meaning and you can do that by writing clean and Semantic HTML code. So, if you're not sure whether some piece of code is wrong or not, check it first.

HTML Version

Currently, the standard HTML version on the web is the fifth revision of the language (which was completed on October 2014).

The previous version of HTML, the forth one, was released in 1997. However, it was constantly updated as the developer community needed (this is happening with HTML 5 too and it's a natural thing).

The Changes

HTML 5 introduces several new tags, as it removes several others.

In order to follow W3C's HTML specification properly, we have to add a line in our HTML file saying to browser which version are we using. Since HTML 5 is now the default, we declare we are using it by simply writing at the top of our file:

<!doctype html>

This doesn't mean you can't specify and use another HTML revision. Checkout this link to go back to the stone age.

The most inportant tags added to this version are the semantic tags:

<header>, <footer>, <article>, and <section>

This W3C article explains more about the new HTML 5 tags and check out this article if you're wondering how is the web changing right now.(development wise)

Here are some examples for you to understand better HTML.

1 - Simple Paragraph

<!doctype html>
<html>
<head>
  <title>Simple Paragraph</title>
</head>
<body>
  <h1>My Simple Paragraph</h1>
  <p>Some cool text</p>
</body>
</html>

Which becomes:

My Simple Paragraph

Some cool text

2 - Shopping List

<!doctype html>
<html>
<head>

</head>

    <li>Beer</li>

    <li>Milk</li>
    <li>Tooth paste</li>
  </ul>

</html>

My Shopping List

  • Cereal
  • Beer
  • Doritos
  • Milk
  • Tooth paste

3 - Simple Form

<!doctype html>
<html>
<head>
  <title>Simple Form</title>
</head>
<body>
  <h1>My Simple Form</h1>
  <form>
    <label>First Name:</label>
    <input type="text"/> <!-- This is a self closing tag (the use of / is optional) --> 
    <br/><br/> <!-- This line inserts two line breaks to your content --> 
    <label>Last Name:</label>
    <input type="text"/>
  </form>
</body>
</html>

HTML "id" and "class" Attributes

The most common attributes we'll be using on HTML programming are the id and the class attributes.

There are several situations where you'll need these attributs. For now, keep in mind that when you set the class and/or the id for an element, the DOM will now it, and you can later on find your element by the class or the id you've chosen.

The id attribute is used to a single element on the DOM and the programmer is not allowed to make multiple repetitions of the id. If you do this, no error will be displayed, but it's semantically wrong.

The class attribute, however, can be used several times throughout the HTML. It's useful for making elements to be grouped and inherit the same features.

You will understand better the importance of these attributes when we start with CSS and JavaScript.
