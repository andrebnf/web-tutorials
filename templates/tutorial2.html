CSS Quick Guide

It's highly recommended that you read this article before continuing. It contains all the concepts that we'll use on our project (and it will be updated constantly). This article also contains Useful Links for you to learn and understand better CSS.

Beggining with CSS

In order to make our page more pretty, we'll use some CSS code. If you have no clue what CSS is, just check this quick guide.

Creating the Stylesheet

To create the styesheet, navigate to the left side on the NetBeans window, right click on Web Pages and navigate through: New > Other and under the Web category, look for Cascading Style Sheet on the right side of the window.

Refering the CSS on the Index Page

In order for our CSS content to be read by the index file, we have to refer the style.css file on our index.jsp. To do this, add the following line to the <head> element on the .jsp file.

<link rel="stylesheet" type="text/css" href="style.css">

Now we're ready to go.

Positioning Elements on the Page

General Settings

Feel free to erase any content on the CSS file if you wish.

Let's start the styling by setting a minimum width to the page and getting rid of the margins:

body {
  min-width: 500px;
  margin: 0;
}

Now, let's position the elements <header>, <section>, and <footer> on the middle of the page. Notice here that we want to change styles for three elements, and the style will be the same for all of them. Instead of refering each one of the elements alone on the CSS, we can set a class and, back on our index.jsp file, we can say that these elements have such a class. We'll call this class container.

So let's get back to our index.jsp page and add the following classes:

<body>
  <header class="container">
    ...
  </header>
  <section class="container">
    ...
  </section>
  <footer class="container">
    ...
  </footer>
</body>

Now we can set the style for the content class by doing:

.container {
  width: 85%; /* If the width is 85%, we'll still have 15% of the page outside the container */
  padding: 0 7.5%; /* 0 refers to top and bottom paddings and 7.5% to left and right paddings */
  clear: both; /* Prevents that the float property "escapes" to other elements */
}

The Header

Now that our elements are correctly positioned on the page, let's position the elements inside the <header>.

We want the header to have only one line, the navigation menu to be on the right side and the title to be on the left side. Let's also change the title size the margin of it's boundering box. To do this, add the following to the CSS:

header {
  height: 50px; /* Size of the header */
}
header h1 a { /* Selecting the link element inside de h1 */
  text-decoration: none; /* Removes the underline */
}
header h1 { /* Selecting the h1 element which is under the header*/
  font-size: 22px;
  margin: 0;
  padding: 10px 14px; /* 10px to both bottom and top paddings and 14px to both left and right paddings */
  float: left; /* Pushes all the h1 block to the left */
}
header nav {
  float: right; /* Pushes all the nav block to the right */
}

Next up, let's redefine the way the menu list is shown by default. We want no "dots" on the side of the items and we also don't wan't any margins or paddings. The following code will do the trick:

nav ul {
  list-style: none;
  margin: 0;
  padding: 0;
}

We have to be careful here. On our page we have only one nav tag. If we had more than one, the properties above would affect them too. We can avoid that by setting an id or a class to the desired element.

Now, let's edit the menu items. We want them to be on the same line and to look like real buttons (not just links). We can do this bu writing on our CSS:

nav ul li {
  display: inline-block; /* All li elements on the nav will be displayed on the same horizontal line */
  margin-top: 15px; /* Gives 15px of distance from the top of the page */
}
nav ul li a {
  text-decoration: none; /* Removes the underline */
  padding: 10px 14px; /* 10px to both bottom and top paddings and 14px to both left and right */
                      /* paddings, giving the link a look of a button */
}

There's an insteresting thing going on here, which is also covered on our quick guide. The nav ul li element here is floating to the right (once one of its parents also is) and that means that we are taking this element off the document flow. When we do this, we lose the padding control of the element and that's why we have to set some top margin to it.

The Form

On the middle of the page we have a <section> which has a <article> and inside this article we have our <form>, which we want to be centered. There is a simple way of doing this:

article {
  width: 400px;
  margin: 0px auto; /* Here, we're saying that this box will have no margin on top and on the bottom*/
                    /* and we're also setting the sides' margin to be automatically calculated equally*/
  margin-top: 50px; /*Giving some space on the top of the article*/
  padding: 0px 20px; /* Setting 0px of padding on the top and on the bottom and padding of 20px on the sides */
}
form {
  text-align: center;
}

The Footer

The idea here is to make the footer fixed on the bottom of the page. This concept is called sticky footer and we can achieve this by giving this element a position property (for a more complete understanding on the CSS Advanced Positioning techniques, check out the quick guide.)

footer{
  position: absolute; /* The element has an absolute position inside its container */
  bottom: 0px; /* Distance from the bottom */
  text-align: center;
  width: 100%;
}

We also have to add a property to the <body> specifying that his minimum height will be 100% of the page's height. We also have to set the height of the <html> element to 100%. This means that even thought the <html>'s height is limited by being 100%, the <body>'s height will strech as it gets larger contents.

The following items should be on your CSS:

html {
  height: 100%;
}
body {
  min-width: 500px;
  margin: 0;
  padding: 0;
  min-height: 100%;
  position: relative;
}

Awesome! Now all the content is correctly positioned on the page. Let's move on and give it some color.

Adding Color and Extra Styles to the Page

Styling the Header and the Footer

The idea is to have the <header> and the <footer> on the same color. Also, we want the top right links to look like buttons, so let's set some border on them. Let's also set a salmon color on the background and set all the foreground text to white. Here's the code:

header, footer {
  background-color: salmon;
  color: white;
}
header a, /* General link tag */
header a:link, /* Links that have not been visited */
header a:visited /* Links that have already been visited */,
header a:active { /* Behaviour triggered when the link is clicked */
  color: white;
}
nav ul li a {
  border: 1px solid white;
}

Putting the Form on a Bordered Box

Now, let's put our <form> inside a box where the background will remain the same, but we'll add some rounded border to it. Just do the following:s

article {
  border: solid 1px #999;
  -webkit-border-radius: 12px; 
     -moz-border-radius: 12px; 
          border-radius: 12px;
}

At the piece of code above, we are using some Vendor Prefixes, such as -webkit- and -moz-. This is needed for browser versions that don't have a native implementation of the border-radius property. The -moz- prefix is for Mozilla older browsers and the -webkit- is used for Apple Safari, webkit based Google Chrome and some others. Browsers that have the native property implemented will simply understand the last line (with no prefix at all).

Changing the Main Font

The standard font most of the browsers uses is "Times New Roman". However, we can set up a more rounded (or sans-serif) font, which will make our application to look nicer. For this example, we'll change it to Tahoma.

To do that, we will use the font-family CSS property inside the <body>:

body {
  /* ... */
  font-family: "Tahoma", sans-serif;
}
