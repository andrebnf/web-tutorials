Intro

We know that the base of a web page is writen in HTML, but this language by itself doesn't provide a nice-looking layout. We need something to edit color, positioning, fonts and more. This is where CSS comes in.

CSS stands for Cascading Style Sheets and basically it provide tools for editing styles on your page.

Every element (tag) is accessible through CSS. Besides that, that are pseudo elements that can also be changed using CSS. But this a subject for another article.

Ways of Setting CSS to Elements on the Page

Say we have the following code and we want to change it's background color:

<!doctype html>
<html>
<head>
  <title>Page Title</title>
</head>
<body>
  <h1>Hello Web</h1>
</body>
</html>

There are three ways of doing it and two of them use CSS Selectors to do so.

Inline CSS

One way to this is by using a HTML attribute called style inside the <body> (which is the part of the HTML that fills the document). So if we just add the CSS property called background-color with the value blue, our background will have a blue color. Here's what we do:

<body style="background-color: blue;">
  <h1>Hello Web</h1>
</body>

Now the background is blue! Did you notice how the style was set? This is the CSS pattern:

css-property: its value;

This is basic way to refer a HTML element to some style. All you have to do is look for the element and just add an attribute. But some elements can have several CSS properties and this is not a good way to organize it.

In-document CSS Block

<!doctype html>
<html>
<head>
  <title>Page Title</title>
	
  <style>
    body {
      background-color: blue;
    }
  </style>

</head>
<body>
  <h1>Hello Web</h1>
</body>
</html>

This is one way to organize better your styles. Instead of writing individual pieces of CSS for each element you want to stylize, you create a element called <style> inside the HTML file (usually it comes nested in the <head> element.

Well, I see a problem here. Imagine you have a header navigation menu on every page of your application. You'll have to write (or copy and paste) the code on every HTML file in order to get the navigation menu to work as you want. This makes this concept not reusable and it's not recommended for large applications.

CSS File

In order to make your CSS code more reusable and readable, you can create another file with all your styles on it.

Now, let's get our CSS code from the last example and put it on a file, which I'll call styles.css:

body {
  background-color: blue;
}

All we have to do now is to tell the HTML file that we have some style already set and that they're on a file under the same directory with the name of style.css.

In this case, I'll have a HTML file named index.html with the content:

<!doctype html>
<html>
<head>
  <title>Page Title</title>
	
  <link rel="stylesheet" type="text/css" href="style.css">

</head>
<body>
  <h1>Hello Web</h1>
</body>
</html>

That makes the organization easier when it comes to styling a big project. You can even separate the CSS files in folders. For that, keep in mind that you'll have to refer the relative path to the file on the href attribute starting from the HTML root path. To refer a path here just use UNIX-like syntax for path definition. So if your CSS file is one level higher on your folder structure, just add ../ before the file name on the href attribute and you should be fine.

CSS Selectors

CSS uses the DOM tree to find objects in the document so the browser knows to what element is the CSS property refering to. There are plenty of CSS Selectors and basically you can refer any element on the document.

Refering Element Tags

Let's begin by selecting basic HTML element tags. To do this, on our css code, we just have open a new block begining with the tag name and followed by brackets, like the following:

div {
  height: 300px;
}

On this example, we are saying that every <div> element will have a height of 300px;

Using an Id or a Class to Select an element

The most secure and organizable way to select an element via CSS, is to set some id or class values to the HTML element, as shown on the HTML Quick Guide.

Both id's and class' references on CSS will be, respectively: 

#brand-icon { /* # character for id */
  font-size: 20px;
}
.text-center { /* . character for class */
  text-align: center;
}

Using Attributes to Select an Element

You can also select an element on CSS by specifying a certain attribute, or even a specific value. Here's the example:

HTML:

<html>
  ...
  <body>
    ...
    <input type="text">
    <input type="password" name="passcode">
    ...
  </body>
</html>

CSS:

input { /* Selects both input fields */
  width: 100%;
}    
input[type="text"] { /* Selects only first input */
  color: darkblue;
}
input[name] { /* Selects only second input, since the first one doesn't have such attribute */
  color: purple;
}

Selecting Children

Since the CSS uses the DOM to find the elements, we can specify a chain (or a hierarchy) in order to reach the desirable element an then to style it as we wish. Let's consider the following HTML:

<html>
  ...
  <body>
    ...
    <p>Some text outside the div</p>
    <div>
      <p>Some text inside the div</p>
    </div>
    ...
  </body>
</html>

Now, let's say we want to set a different style to both paragraphs. In order to do that, we need to select them separately:

p { /* Selects both paragraphs */
  font-size: 17px;
}    
div p { /* Selects only the second paragraph */
  color: darkblue;
  font-size: 20px;
}

Notice that we are not selecting the first paragraph distinctly. Rather, we are setting the style to both paragraphs and overriding the second paragraph: since its styles were set right after both paragraphs have been set, the CSS will keep the last change made, which refers only to the second.

Combining Selectors

Keep in mind that you can, at any moment, combine these selectors above. So, it's common to have something like:

header div #login-btn {
  color: seagreen;
}    
div .container .foo p.small-text { 
  font-size: 10px;
}

Common Mistakes Regarding Selectors

The main issue here is when you can't refer the desired element. Unfortunately, there is no secret on this matter: always keep you code organized and set ids and classes to keep you organized.

When styling a page, you also have to be concerned about the excess of organization. On big projects, if you have a class or an id for each element, it's easy to get lost.

The most trivial step you should make, is to be sure that your CSS doesn't have any typo, misspelling or syntax errors.

A typo can be considered an error on your code that still works, but no the way you wanted it to. An example of this is our last case:

div .container .foo p.small-text { 
  font-size: 10px;
}

Have you noticed that there is something different of this selector? It starts with a div, which has a child with a class container. The container element has a child with a foo class, which has a paragraph. On this code, we are saying that the paragraph has a small-text class, and that's indicated by simply not separating the <p> and the class name. The following code is slightly different:

div .container .foo p .small-text { /* Space added between "p" and ".small-text" */
  font-size: 10px;
}

Before we was setting the property font-size to a paragraph which had a small-text class. Now, we are setting the property to the paragraph's child (or children) whose class is small-text.
