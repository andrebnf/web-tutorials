HTML Quick Guide

It's highly reccomended that you read this article before continuing. It contains all the concepts that we'll use on our project (and it will be updated constantly). This article also contains Useful Links for you to learn and understand HTML better.

Begining with NetBeans

On these tutorials we will use NetBeans IDE version 8.0.2 to manage our application. Although it's not recommended, you can use any tool you want to test and run your application. In that case, make sure you use Apache Tomcat to simulate a server on your local machine, so when you download or exchange file with your classmates, there won't be any trouble.

The major change on NetBeans 8 from the previous versions is the use of the Java EE Web version 7. The Tomcat version used on the example is 8.0.15.0. Make sure you create your projects on these conditions.

Creating the Project

To begin with, create a new project on NetBeans by navigating to File > New Project or simply by pressing CTRL + SHIFT + N

Now, select Web Application under Java Web

Click on Next.

Now, choose a name to your project and change the folder where it will be saved, if you will. If the option Use Dedicated Folder for Storing Libraries is not checked, do it. (optional)

Checking this option is a convention. It prevents the risk of opening the same project in a different machine and get a missing libraries error.

Now, select Apache Tomcat as our server.

Since we won't use any framework on this project, just click on Next at the following step(s) and click on Finish to generate your project.

Editing the HTML File

When finish the create project wizard, NetBeans will automatically create a file called index.html. However, we'll use some JSP code later on, so let's create a index.jsp file. Obs: NetBeans 7 will create a index.jsp file for you.

To create our file, go to your projects view (usually placed on the left side of the window), expand the project container, right click the Web Pages folder and navigate to New > JSP... and follow the image below:

Now, delete the old index.html by right clicking it and selecting the Delete option. Then click on Yes to confirm it.

The generated .jsp file will look like:

<%-- 
  Document   : index
  Created_on : 31/02/2015, 0:23:18 PM
  Author     : andrebnf
--%>

<%@page contentType="text/html" pageEncoding="UTF-8"%>
<!DOCTYPE html>
<html>
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8">
    <title>JSP Page</title>
  </head>
  <body>
    <h1>Hello World!</h1>
  </body>
</html>

Here we have the base to our Ride Share Web App.

Feel free to erase the commented header (between the <%-- and --%> tags) if you wish.

Preparing the HTML

There are some few things you'll have to change here:

  1. Change the text inside <title> tag so our App name would be shown at the browser's top bar.

  2. Change the <meta> tag in order to update its syntax to the HTML 5 definition.

  3. Erase anything inside the <body> tag so we can start defining our page's structure.

The result is:

<%@page contentType="text/html" pageEncoding="UTF-8"%>
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Ride Share</title>
  </head>
  <body>
  </body>
</html>

Adding the Header and a Navigation Menu

All done? Now let's create a header on our page so we can add an awesome navigation bar to it. All you have to do is add a <header> element inside the body and then add the elements <h1> and <nav> inside the header.

Inside the <nav>, create a simple list with the desired items.

On our case, the result will be:

<header>
  <h1><a href="index.jsp">Ride Share</a></h1> <!-- Our title, which is also a link -->
  <nav>
    <ul> <!-- Our nav menu -->
      <li><a href="index.jsp">Home</a></li>
      <li><a href="#">Sign Up</a></li>
    </ul>
  </nav>
</header>

There are three links on the list above. The first two will take the user to the index.jsp page and the third one is a null link and it will take us nowhere.

We use the "#" charactere preceding the id attribute of an element so when clicked, the link will change the current scroll position of the page to that element. For example, if we have a div at the very bottom of our page with an id of "foo" and, at the top of the page we have a link with the value "#foo" on the href attribute, when this link is clicked, the page will be scrolled to the bottom of the page, positioning the "foo" element on the top of the page.

Adding a Simple Form

The next step is to build a simple form for login purposes. All we want are two fields (username and password) and one submit button.

We'll add the form by using the <form> HTML element. Inside this element, we'll organize the elements inside three paragraphs. The first paragraph will contain a <label> for the username field and a <input> element so the users can actually write their username. The second paragraph will be similar to the first one, but instead of reading the username, it'll get the user's password. The third paragraph will contain a button for the login action to be performed.

All these elements will be grouped on a <section> element for better organization. This section will also have a title. Check this out:

<section>
  <article>
    <h1>Login</h1>
    <form>
      <p><label for="login">Username:</label><input type="text" id="login"></p>
      <p><label for="senha">Password:</label><input type="password" id="senha"></p>
      <p><input type="submit" value="Log me in"></p>
    </form>
  </article>
</section>

There are a few attributes above that are worth of an explanation. First, the "for" attribute of the <label> element refers to the its respective input field "id" attribute. By doing this, we are giving some extra meaning to our form. There are also the "type" attributes of the <input> fields. Those attributes are already defined by W3C (you cannot simply make them up). Click here to check the full reference for the "type" attribute of an input tag.

Adding the Footer

To finish the index page, all we have to do is to place a <footer> element before the body tag is closed. Inside the <footer>, just place some text. Here's what I did:

<footer>
  <p>Web Development - UFSCar Sorocaba - 2015</p>
</footer>

The Complete Index Page

All done! Your index page should look like this.

<%@page contentType="text/html" pageEncoding="UTF-8"%>

<html>
<head>
  <meta charset="UTF-8">
  <title>Ride Share</title>
</head>
<body>
  <header>
    <h1><a href="index.jsp">Ride Share</a></h1>
    <nav>
      <ul>
        <li><a href="index.jsp">Home</a></li>
        <li><a href="#">Sign up</a></li>
       </ul>
    </nav>
  </header>
  <section>
    <article>
      <h1>Login</h1>
      <form>
        <p><label for="login">Username:</label><input type="text" id="login"></p>
        <p><label for="senha">Password:</label><input type="password" id="senha"></p>
        <p><input type="submit" value="Log me in"></p>
      </form>
    </article>
  </section>
  <footer>
    <p>Web Development - UFSCar Sorocaba - 2015</p>
  </footer>
</body>
</html>
