Intro

A essential skill for a Front-End Web Developer or a Web Designer is posining.

Sometimes, you want to create a nice-looking layout and in most of the time, you'll need to know some tricks to do so.

This article presents some techniques that are often used for advanced positioning of elements on the page.

Elements' Boundering Boxes

The first thing to have in mind, is that you element has some boundaries:

  • padding,
  • border,
  • and margin.

These boundaries are external to the content. So when you set the height property on CSS, you are setting the content's heignt only.

The Float Proprerty

This property is very useful to set complext layouts positioning.

What it basically does, it pushes the floating element as far as it's possible to the right or the left. The "possible" here means the container element's limit.

The float property can only have two values: right or left. When you specify one of these two values to the element, you are taking this element away from the page's flow. That means, for some browsers, that it's padding is no longer valid.

When a element is floated, it's placed on the top, left or right, of the containing element and can be later custom positioned by setting some margin to it.

The fact that the float property takes the element out of the page's flow can be dangerours if not used correctly. Let's consider the image below, taken from a W3C's documentation, which illustrates three boxes: one of them is floating to the left and the other are places within the page's flow. The bottom box is correctly placed, but its content is avoiding the floating element, giving an unexpected look to the page. That happens because the horizontal boxes don't even notice the floating element as an obstacle for positioning, since this is floating on the left. However, the horizontal boxes do know that something is floating on the left.

Image source: W3C Visual Formatting Model

The Clear Property

To fix the second box, we can set a property called clear and specify, as a value, which side we want the page's flow to be cleaned. The property and it's value are:

#second-box {
  clear: left; /* options are: left, right, both */
}

The Position Property

A good way to position elements on you page is to use the position property available on CSS. There are a few values to this property:

  • static
  • fixed
  • absolute
  • relative
  • inherit

These values above specify a reference point to the element. After this specification, we can reposition the element by using these properties: left, right, bottom and top.

The static value is set by default on every element and it means that the element will follow the page's flow. If this property is set, the properties left, right, bottom and top will have no effect on the element's current position.

The fixed value tells the element to be positioned accordingly to the browse window. We use this property when we want to set a sticky navbar (the navbar will always be shown don't matter wheter the users scroll or not), for example:

nav {
  position: fixed;
  width: 100%; /* Fits the page horizontally */
  top: 0px; /* Distance from the browse window */
}

The absolute value means that the element will be positioned relative to the first parent which has a position value other than static. For this example, let's say we have a box and we want to add a button on the right of this box:

.box {
  position: relative;
  width: 50%;
  height: 200px;
}
.button {
  position: absolute;
  right: 10px;
  top: 10px;
  width: 70px;
  height: 70px;
}

The result will be:

 .box
 .button

The relative value sets the element to be able to be positioned accordingly to its current position. This property is often used when we have a child of this element that we want to position in a an absolute way.

The example below simulates the margin-top attribute. However, let's not forget that we are repositioning the element outside of the page. Thus, any elements placed after the repositioned one won't be aware of it's actual position.

<html>
  <head>
    ...
    <style>
      div {
        height: 40px;
      }
      div.first {
        position: relative;
        top: 10px;
      }
    </style>
  </head>  
  <body>
    <div class="first"></div>
    <div class="second"></div>
  </body>
</html>

This code will result on:

 .first
 .second

To conclude, the inherit value basically sets the element to have the same property's value as its parent do.

What Should I Use?

As explained here, both methods (floating or the position property) take the element out of the page's flow, so they should be used with some caution.

Case 1 - Float

In regards to what is best to use, it depends on the application. Let's illustrate it here with a column grid: we want to keep three columns inside an element. For that matter, we could use three floating elements with width of 33% each. We could also use three absolutely positioned elements with width of 33% and relative left properties (the first one will have left: 0px, the second one, left: 33% and the third one, left: 66%. Notice that we are not considering the elements to have any border, padding or margin.

On the example above, the best choice is to use floating elements. As an argument, let's say we want to add another element after the grid. If we use the clear: both property on this element, everything will act as we expect and the element would be add after the grid. However, if we have three elements on the grid with the position: absolute property set, the following element wouldn't now where to be exactly placed, and it will sit on the top of the element.

The example to this case is the following:

<html>
  <head>
    ...
    <style>
      .floating-grid {
        width: 100%;
      }
      .floating-grid .item {
        height: 40px;
        width: 33%;
        float: left;
      }
      .floating-grid .another {
        clear: both;
        width: 100%;
        height: 40px;
      }
      .absolute-grid {
        position: relative;
      }
      .absolute-grid .item {
        position: absolute;
        width: 33%;
        left: 0px;	
      }
      .absolute-grid .item:nth-child(2){
        left: 33%;  
      }
      .absolute-grid .item:nth-child(3){
        left: 66%;  
      }
      .absolute-grid .another{
        width: 100%;
        height: 40px;
      }
    </style>
  </head>  
  <body>
    <p>Floating Items Grid:</p>
    <div class="floating-grid">
    	<div class="item"></div>
    	<div class="item"></div>
    	<div class="item"></div>
    	<div class="another"></div>
    </div>
    <p>Absolute Items Grid:</p>
    <div class="absolute-grid">
    	<div class="item"></div>
    	<div class="item"></div>
    	<div class="item"></div>
    	<div class="another"></div>
    </div>
  </body>
</html>

Floating Items Grid:

 .floating-grid.item
 .floating-grid.item
 .floating-grid.another

Absolute Items Grid:

 .absolute-grid.item
 .absolute-grid.item

Something has gone wrong on the example above. And that's because the absolute-grid.another element is not understanding the page's flow. Here, it's better to use float.

Case 2 - Some Differences

When it comes to positioning, the most dangerous and failing case it's when you set an absolute or a fixed position, which takes the elements totally off the page flow. Thus, it's recommended for designers to always use the position element to do simple tasks (as putting an element on the side of a div in order to make it look like a button). If we used the float option for that matter, the preceding and subsequent elements would wrap around this button, producing a potencially undesired behaviour of the layout.

Let's examplify this case: we want a button on the right side of a box. This button will be placed over the text and it will have a opaque background. Here's the code

<html>
  <head>
    ...
    <style>
      .box {
        position: relative; 
        width: 50%;  
        height: 200px;
      }
      .absolute-button {
        position: absolute;
        right: 10px;
        top: 10px;
      }
      .floating-button {
        margin: 10px;
      }
    </style>
  </head>  
  <body>
    <p>Absolute Position on the Button:</p>
    <div class="box">
    	<div class="floating-button"></div>
    	<p>SOME TEXT</p>
    </div>	
    <p>Float Position on the Button:</p>
    <div class="box">
    	<div class="floating-button"></div>
    	<p>SOME TEXT</p>
    </div>
  </body>
</html>
